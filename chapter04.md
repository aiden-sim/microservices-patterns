# 트랜잭션 관리 : 사가
- 마이크로서비스 아키텍처에서도 단일 서비스 내부의 트랜잭션은 ACID가 보장하지만, 여러 서비스의 데이터를 업데이트하는 트랜잭션은 구현하기 까다로움
- 분산 트랜잭션 기법은 요즘 잘 안맞음
- 사가를 써야 하는데 ACD(원자성, 일관성, 지속성)만 지원하고 격리가 안되서 영향을 줄일 수 있는 설계 기법을 적용해야 됨
